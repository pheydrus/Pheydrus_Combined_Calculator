# Functional Requirements: Vercel Serverless API

## Overview

A Vercel serverless function that acts as a secure proxy between the client-side chat interface and the Anthropic Claude API. The endpoint accepts chat messages with knowledge base context, streams Claude's response back to the client via Server-Sent Events (SSE), and ensures the API key is never exposed to the browser.

## Scope

- Single POST endpoint: `/api/chat`
- Server-side Claude API integration using `@anthropic-ai/sdk`
- SSE streaming response for real-time token delivery
- System prompt engineering for Pheydrus brand voice
- Input validation and error handling
- Compatible with Vercel Hobby plan (10-second timeout, streaming keeps connection alive)

## User Stories

### US-1: Send Chat Message with Context

**As a** chat interface
**I want** to send user messages along with relevant knowledge base context to the API
**So that** Claude can generate informed, context-aware responses

**Acceptance Criteria:**

- [ ] POST `/api/chat` accepts a JSON body with `messages` and `context` fields
- [ ] `messages` is an array of chat history objects with `role` ("user" or "assistant") and `content` (string)
- [ ] `context` is an array of knowledge base chunks with `id`, `title`, `content`, and `category` fields
- [ ] The API constructs a system prompt that includes the provided context
- [ ] Claude receives the full conversation history plus system prompt
- [ ] Response begins streaming within 2 seconds of request receipt

**Example Request:**

```json
{
  "messages": [{ "role": "user", "content": "What is my life path number used for?" }],
  "context": [
    {
      "id": "life-path-numbers_chunk_0",
      "title": "life_path_numbers",
      "content": "Life path numbers reveal your soul's purpose...",
      "category": "Public_CMO"
    }
  ]
}
```

### US-2: Stream Response Tokens via SSE

**As a** chat interface
**I want** to receive Claude's response as a stream of tokens
**So that** the user sees the response being typed out in real time

**Acceptance Criteria:**

- [ ] Response uses `Content-Type: text/event-stream`
- [ ] Each SSE event contains a `data` field with a JSON payload: `{"token": "word"}`
- [ ] Tokens are sent as they are generated by Claude (no buffering)
- [ ] A final SSE event with `data: [DONE]` signals the end of the stream
- [ ] The stream keeps the Vercel connection alive beyond the 10-second timeout
- [ ] Client can read the stream using the Fetch API's `ReadableStream`

**Example SSE Stream:**

```
data: {"token":"Life"}

data: {"token":" path"}

data: {"token":" numbers"}

data: {"token":" are"}

data: [DONE]
```

### US-3: Pheydrus Brand Voice System Prompt

**As a** product owner
**I want** Claude to respond as a warm, knowledgeable Pheydrus guide
**So that** users receive on-brand, helpful responses grounded in Pheydrus content

**Acceptance Criteria:**

- [ ] System prompt instructs Claude to use ONLY the provided context to answer questions
- [ ] Claude cites sources using `[Source: filename.txt]` format inline
- [ ] Claude stays on-topic: Pheydrus content, astrology, numerology, personal development
- [ ] Off-topic questions receive a polite redirect to relevant Pheydrus topics
- [ ] Tone is conversational but informative (warm, encouraging, not clinical)
- [ ] Responses default to 2-4 paragraphs unless the user explicitly requests more detail
- [ ] Claude acknowledges when it doesn't have enough context to answer fully
- [ ] Claude does not fabricate information not present in the provided context

### US-4: Secure API Key Handling

**As a** developer
**I want** the Anthropic API key stored securely on the server
**So that** it is never exposed to the client browser

**Acceptance Criteria:**

- [ ] API key is read from `ANTHROPIC_API_KEY` environment variable
- [ ] API key is never included in client-side bundles
- [ ] API key is never returned in API responses or error messages
- [ ] Missing API key returns a 500 error with a generic message (no key details)
- [ ] API key is configured in Vercel project settings for production
- [ ] API key is configured in `.env.local` for local development (file is gitignored)

### US-5: Input Validation

**As a** developer
**I want** the API to validate incoming requests
**So that** malformed or oversized requests are rejected before calling Claude

**Acceptance Criteria:**

- [ ] Missing `messages` field returns 400 with error message
- [ ] Empty `messages` array returns 400 with error message
- [ ] Individual message content exceeding maximum length (10,000 characters) returns 400
- [ ] Total context content exceeding maximum size (100,000 characters) returns 400
- [ ] Invalid `role` values (not "user" or "assistant") return 400
- [ ] Non-POST requests return 405 Method Not Allowed
- [ ] Request body exceeding 1 MB returns 413 Payload Too Large

### US-6: Error Handling

**As a** developer
**I want** the API to handle errors gracefully
**So that** the client receives meaningful error responses

**Acceptance Criteria:**

- [ ] Missing ANTHROPIC_API_KEY: returns 500 `{"error": "Chat service is not configured"}`
- [ ] Claude API rate limit: returns 429 `{"error": "Service is busy, please try again"}`
- [ ] Claude API error: returns 502 `{"error": "Failed to generate response"}`
- [ ] Invalid request body (not JSON): returns 400 `{"error": "Invalid request format"}`
- [ ] All error responses use JSON format with an `error` field
- [ ] Error responses never expose internal details (API keys, stack traces)

### US-7: Local Development Support

**As a** developer
**I want** the API to work with `vercel dev` locally
**So that** I can test the full chat flow without deploying

**Acceptance Criteria:**

- [ ] API works with `vercel dev` command for local testing
- [ ] `.env.local` file with `ANTHROPIC_API_KEY` is loaded automatically
- [ ] Streaming works correctly in local development mode
- [ ] CORS headers are set appropriately for local development
- [ ] No code changes required between local and production

## System Prompt Specification

The system prompt follows this structure:

```
You are Pheydrus AI, a warm and knowledgeable guide for personal development,
astrology, and numerology. You help users understand their cosmic blueprint
and navigate their personal growth journey.

INSTRUCTIONS:
- Answer questions using ONLY the provided context below
- If the context doesn't contain enough information, say so honestly
- Cite your sources inline using [Source: filename] format
- Stay on topic: astrology, numerology, personal development, and Pheydrus offerings
- Keep responses concise (2-4 paragraphs) unless the user asks for more detail
- Be warm, encouraging, and conversational in tone
- Never fabricate information not present in the context

CONTEXT:
{formatted context chunks with titles and categories}
```

## Claude Model Configuration

| Parameter   | Value             | Rationale                                              |
| ----------- | ----------------- | ------------------------------------------------------ |
| Model       | claude-sonnet-4-6 | Balance of quality and speed for chat                  |
| Max tokens  | 1024              | Keeps responses concise, sufficient for 2-4 paragraphs |
| Temperature | 0.7               | Conversational warmth without hallucination risk       |
| Stream      | true              | Required for SSE delivery                              |

## Performance Requirements

- First token delivered within 2 seconds of request receipt
- Streaming keeps Vercel connection alive (no 10-second timeout with streaming)
- API cold start under 3 seconds
- No request queuing or batching (one request = one Claude call)

---

**Important:** The API key must NEVER appear in client-side code, error messages, or logs. The system prompt is the primary control for response quality and brand voice.
